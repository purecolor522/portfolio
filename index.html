工作。

釐清問題，定義方向

從使用者訪談、需求排序到 PRD 與 roadmap，把利害關係人的雜訊收斂成一條清楚的產品路線。

把想法視覺化

品牌識別、主視覺、簡報與 UI。需要時能自己把畫面做出來，不必等設計排程，溝通更快也更準。

用 AI 加速交付

串接 API 與低程式工具，做出小工具與內部自動化，把重複的工作變成一次設定就完成。

// about

設計背景出身，
在做產品的路上學會用 AI。

我從平面設計起步，後來轉做產品。這讓我習慣同時用兩種眼睛看事情：使用者看到的畫面，和背後要解決的問題。

這幾年我把 AI 變成日常工具，做反饋分析、競品摘要、營運看板。對我來說，能自己快速做出原型，比一份完美的規格更能推動產品往前。
